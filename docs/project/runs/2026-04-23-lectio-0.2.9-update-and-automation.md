## Automation Notes

- Dependabot now watches the `frontend/` npm manifest so future Lectio releases should arrive as PRs instead of requiring manual version checks.
- `package-lock.json` is the CI-authoritative lockfile because frontend CI uses `npm ci`.
- `pnpm-lock.yaml` is still refreshed locally for parity, but Dependabot will not keep that file in sync by itself.
- A dedicated GitHub Action now warns on PRs if `frontend/pnpm-lock.yaml` drifts from the npm lockfile state.

## Risks / Follow-up

- If the team starts relying on `pnpm-lock.yaml` as a source of truth, we should add a second sync step or switch to a single lockfile workflow.
