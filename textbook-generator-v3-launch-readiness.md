# Textbook Generator v3 — Launch Readiness Proposal

## Purpose

The v3 pipeline is built end-to-end. This proposal covers the remaining
gaps required to run a real teacher brief through the full system and
produce a rendered, downloadable resource.

**Scope:** Lectio version bump · manifest injection into Lesson Architect ·
lens library · navigation · v3 PDF export endpoint + print view

**Out of scope:** Coherence reviewer tuning · persona example JSON files ·
staging environment · auth changes

**Codebase:** Lectio + Textbook Generator backend + Textbook Generator frontend

**Note:** `V3LectioSectionEmbed` is already fully implemented — it imports
from `lectio`, uses `templateRegistryMap` and `LectioThemeSurface`, and
renders from `mergedFields`. No changes needed there.

## Gap 1 — Lectio: bump version and re-export contracts

### What and why

The Lesson Architect reads `manifest.json` from the Lectio contracts
directory. This file is already generated by `npm run export-contracts`
and already structured correctly with `phase`, `role`, `cognitiveJob`,
and `capabilities` per component.

The backend must have the latest contracts. The consumer backend installs
Lectio contracts from GitHub Packages — a new version tag triggers the
publish pipeline and Renovate Bot opens a PR in the Textbook Generator.

### Tasks

```
Lectio repo:
  1. Run: npm run export-contracts
     Verify manifest.json is present in contracts/ and non-empty
  2. Bump package.json version (e.g. current → next patch)
  3. Commit, push, tag:
       git tag v{new-version}
       git push origin v{new-version}
     GitHub Actions publishes to GitHub Packages automatically

Textbook Generator repo:
  4. Merge the Renovate Bot PR that updates the Lectio dependency
  5. Run: uv run python tools/update_lectio_contracts.py
     Verify contracts/manifest.json exists and contains component entries
```

### Verification

```
□ contracts/manifest.json exists in Textbook Generator backend
□ manifest.json contains at least 10 components with phase + role fields
□ npm run export-contracts completes without errors in Lectio
```

## Gap 2 — Backend: inject manifest into Lesson Architect

### What and why

`ARCHITECT_SYSTEM` in `backend/src/generation/v3_studio/prompts.py`
currently has no component list. The Lesson Architect reasons from the
brief alone and may select component slugs that don't exist in Lectio.

The manifest must be loaded from `contracts/manifest.json` and injected
into the system prompt so the architect knows exactly which components
exist, what phase they belong to, and what each one is for.

```python
from pipeline.contracts import get_manifest


def _format_manifest_for_prompt() -> str:
    """Format manifest.json into a compact component list for the architect."""
    manifest = get_manifest()
    lines = ["Available Lectio components (use only these slugs):"]
    for phase_group in manifest.get("phases", []):
        phase_num = phase_group.get("phase", "?")
        phase_name = phase_group.get("name", "")
        lines.append(f"\n  Phase {phase_num} — {phase_name}:")
        for component_id in phase_group.get("components", []):
            component = manifest.get("components", {}).get(component_id, {})
            role = component.get("role", "")
            cognitive_job = component.get("cognitiveJob", "")
            lines.append(
                f"    {component_id}: {role}"
                + (f" [{cognitive_job}]" if cognitive_job else "")
            )
    return "\n".join(lines)


def build_architect_system_prompt() -> str:
    manifest_block = _format_manifest_for_prompt()
    return f"""You are a lesson architect. Output ONLY a valid ProductionBlueprint matching the schema.

{manifest_block}

Rules:
- Only use component slugs from the list above. Never invent new slugs.
- metadata: version "3.0", title, subject (from teacher subject)
- lesson: lesson_mode first_exposure|consolidation|repair, resource_type lesson|mini_booklet
- applied_lenses: min 1 lens with lens_id and effects (non-empty strings)
- voice: register (simple|balanced|formal etc), optional tone
- anchor: reuse_scope string
- sections: min 1, each section_id, title, role, visual_required bool,
  components min 1 with component slug from the list above and content_intent
- question_plan: min 1 items with question_id, section_id, temperature
  warm|medium|cold|transfer, prompt, expected_answer, diagram_required
- visual_strategy: visuals list (can match sections needing visuals)
- answer_key: style string (e.g. concise_steps, answers_only)
- teacher_materials, prior_knowledge lists allowed
"""
```

## Gap 3 — Backend: write the lens library

### What and why

The Lesson Architect currently has no pedagogical lens guidance. It plans
from teacher signals and the component list alone. The lens library gives
it the accumulated teaching wisdom — when to use anchor examples, how to
sequence difficulty, how to adapt for EAL learners, what repair means
vs re-teaching, and so on.

### File structure

```
backend/resources/lenses/
  lesson/
    first_exposure.yaml
    consolidation.yaml
    repair.yaml
    retrieval.yaml
    transfer.yaml
  support/
    high_load.yaml
    eal.yaml
    advanced.yaml

backend/src/v3_lenses/
  __init__.py
  schema.py       # LensSchema Pydantic model
  loader.py       # loads all YAML files
  registry.py     # get_all(), get_by_id(), get_by_category()
```

### Lens schema

```python

from pydantic import BaseModel
from typing import Literal

class LensSchema(BaseModel):
    id: str
    label: str
    category: Literal["lesson", "support"]
    applies_when: str
    reasoning_principles: list[str]
    blueprint_effects: dict[str, str]
    principles: dict[str, str]
    avoid: list[str] = []
    conflicts_with: list[str] = []
```

### Lens YAML files

**`resources/lenses/lesson/first_exposure.yaml`:**
```yaml
id: first_exposure
label: First Exposure
category: lesson
applies_when: >
  The learner has no working model of this concept.
  This is genuine first teaching, not review or practice.

reasoning_principles:
  - Create need before formal explanation.
  - Use one stable anchor example throughout.
  - Model before independent practice.
  - Move warm to medium before attempting cold.
  - Never ask a learner to practise what they have not first understood.

blueprint_effects:
  visual_density: medium_or_high
  practice_progression: warm_to_medium
  vocabulary_policy: define_first
  anchor_reuse: entire_resource
  cold_questions: avoid_for_below_average

principles:
  anchor: >
    Choose one concrete example and reuse it everywhere — explanation,
    worked example, and the first practice questions. Familiarity before
    variation. Consistency reduces cognitive load on first exposure.
  sequence: >
    Create need before building knowledge. The sequence is not negotiable
    on first exposure: orient → explain → model → practise.
  visual: >
    Spatial and procedural concepts need visual support. Reuse the anchor
    shape across visuals. Step-sequence visuals show one step at a time.
  difficulty: >
    Warm questions use the anchor with minimal variation. Medium introduces
    one dimension of variation. Do not reach cold for below-average groups
    on first exposure.
  adaptation: >
    Below average: reuse anchor throughout, whole numbers only, more worked
    steps, diagrams on all questions. Average: anchor in explanation and
    first two questions. Above average: anchor in explanation only, varied
    contexts, transfer question required.

avoid:
  - assuming prior fluency with this concept
  - starting with cold or transfer questions
  - different examples across explanation and practice

conflicts_with:
  - consolidation
  - transfer
```

```yaml
category: lesson

  - Start at medium difficulty, not warm.

    Cold and transfer are the target — that is what consolidation means.
  scaffolding: >
    Reduce scaffolding compared to first exposure. Learner should be
    attempting independently with minimal worked reference.

avoid:
  - re-teaching the concept from scratch
  - using the same examples as the original teaching lesson
  - starting with warm questions unless learner signals are very weak

conflicts_with:
  - first_exposure
  - repair
```

**`resources/lenses/lesson/repair.yaml`:**
```yaml
id: repair
label: Repair
category: lesson
applies_when: >
  The learner has a specific misconception or gap.
  They have been taught before but something went wrong.

reasoning_principles:
  - Target the fault line only. Not the whole concept again.
  - Use a different approach and different example from original teaching.
  - Tight practice set targeting only the misconception.
  - An exit check is required to confirm the repair worked.
  - Do not broaden scope under any circumstances.

blueprint_effects:
  scope: fault_line_only
  examples: different_from_original
  exit_check: required
  visual_strategy: concrete_before_symbolic

principles:
  fault_line: >
    The teacher has identified what specifically went wrong. The lesson
    addresses only that. Teaching is NOT reteaching the whole concept.
  approach: >
    Use a different visual or concrete approach from what failed.
    If the original used symbolic notation, use a diagram first.
    If the original used one analogy, use a different one.
  practice: >
    Questions target only the misconception. Do not include broad
    coverage questions — they dilute the repair.
  exit: >
    Include a short check at the end. The teacher needs to know if the
    repair worked before the next lesson.

avoid:
  - reteaching the whole concept
  - using the same approach as the original failed lesson
  - adding broad practice beyond the fault line

conflicts_with:
  - consolidation
  - first_exposure
```

### Lens loader

```python
# backend/src/v3_lenses/loader.py
from __future__ import annotations
import logging
from pathlib import Path
import yaml
from v3_lenses.schema import LensSchema

logger = logging.getLogger(__name__)
_LENSES_DIR = Path(__file__).parents[2] / "resources" / "lenses"
_REGISTRY: dict[str, LensSchema] = {}


def load_all_lenses(lenses_dir: Path = _LENSES_DIR) -> dict[str, LensSchema]:
    registry: dict[str, LensSchema] = {}
    for path in sorted(lenses_dir.rglob("*.yaml")):
        with path.open(encoding="utf-8") as f:
            raw = yaml.safe_load(f)
        lens = LensSchema.model_validate(raw)
        registry[lens.id] = lens
        logger.info("Loaded lens: %s (%s)", lens.id, lens.category)
    if not registry:
        raise RuntimeError(f"No lenses found in {lenses_dir}")
    return registry


def get_all_lenses() -> list[LensSchema]:
    if not _REGISTRY:
        _REGISTRY.update(load_all_lenses())
    return list(_REGISTRY.values())


def get_lens(lens_id: str) -> LensSchema:
    if not _REGISTRY:
        _REGISTRY.update(load_all_lenses())
    return _REGISTRY[lens_id]
```

## Gap 5 — Backend + Frontend: v3 PDF export

### What and why

v2 PDF renders `/textbook/{generation_id}?print=true&token=...` via
Playwright. v3 needs the same mechanism pointing at a v3 print route.
The Playwright infrastructure, cover page, TOC, and assembly pipeline
are all reused — only the render URL and the endpoint path change.

### Backend changes

**`backend/src/generation/v3_studio/router.py`** — add PDF export route:

```python
from generation.pdf_export.service import PDFExportRequest, export_generation_pdf
from generation.pdf_export.cleanup import cleanup_files
from fastapi import Request
from fastapi.responses import FileResponse
from core.config import get_settings
from core.auth.jwt import JWTHandler, get_jwt_handler

@v3_studio_router.post("/generations/{generation_id}/export/pdf")
async def export_v3_pdf(
    generation_id: str,
    body: PDFExportRequest,
    request: Request,
    current_user: User = Depends(get_current_user),
    jwt_handler: JWTHandler = Depends(get_jwt_handler),
):
    # Verify the generation belongs to this user
    stored = await v3_studio_store.get_generation_owner(generation_id)
    if stored != current_user.id:
        raise HTTPException(status_code=404, detail="Generation not found")

    auth_token = jwt_handler.create_access_token(current_user.id, current_user.email)

    # v3 print route differs from v2
    settings = get_settings()
    v3_settings = settings.model_copy(
        update={
            "pdf_render_base_url": settings.pdf_render_base_url
        }
    )

    result = await export_generation_pdf(
        generation=_make_v3_generation_stub(generation_id, current_user.id),
        document=None,            # v3 does not use PipelineDocument
        auth_token=auth_token,
        request=body,
        settings=v3_settings,
        request_id=getattr(request.state, "request_id", None),
        render_path=f"/studio/print/{generation_id}",  # v3-specific render path
    )

    async def _cleanup():
        cleanup_files(result.cleanup_paths)

    return FileResponse(
        path=result.pdf_path,
        media_type="application/pdf",
        filename=result.filename,
        background=BackgroundTask(_cleanup),
        headers={
            "X-Page-Count": str(result.page_count),
            "X-File-Size": str(result.file_size_bytes),
        }
    )
```

### Frontend: print route

**New file: `frontend/src/routes/studio/print/[id]/+page.svelte`**

A minimal print-mode page that loads the v3 generation canvas and sets
`data-generation-complete="true"` when all sections are ready (Playwright
waits for this selector before generating the PDF):

```svelte
<script lang="ts">
  import { page } from '$app/stores';
  import { onMount } from 'svelte';
  import V3Canvas from '$lib/components/studio/V3Canvas.svelte';
  import { connectV3StudioGenerationStream } from '$lib/api/v3';
  import { buildCanvasSkeleton, mergeComponentField, mergeDiagramFrame, mergePracticeProblem } from '$lib/studio/v3-canvas';
  import type { CanvasSection } from '$lib/types/v3';

  const generationId = $page.params.id;
  let canvas: CanvasSection[] = [];
  let complete = false;

  onMount(async () => {
    // Load blueprint from session store via a lightweight GET endpoint
    const res = await fetch(`/api/v1/v3/generations/${generationId}/blueprint`, {
      headers: { Authorization: `Bearer ${new URL(window.location.href).searchParams.get('token')}` }
    });
    if (!res.ok) return;
    const blueprint = await res.json();
    canvas = buildCanvasSkeleton(blueprint);

    // Replay events from stored generation result
    connectV3StudioGenerationStream(generationId, {
      onComponentReady: (data) => {
        const sid = String(data.section_id ?? '');
        const cid = String(data.component_id ?? '');
        const field = String(data.section_field ?? '');
        const pdata = data.data as Record<string, unknown> ?? {};
        canvas = canvas.map(s => s.id !== sid ? s : {
          ...s,
          mergedFields: mergeComponentField(s.mergedFields, field, pdata),
          components: s.components.map(c => c.id !== cid ? c : { ...c, status: 'ready', data: pdata })
        });
      },
      onVisualReady: (data) => {
        const sid = String(data.attaches_to ?? '');
        const url = typeof data.image_url === 'string' ? data.image_url : null;
        const fi = data.frame_index as number | null ?? null;
        canvas = canvas.map(s => s.id !== sid ? s : {
          ...s,
          mergedFields: mergeDiagramFrame(s.mergedFields, { image_url: url, frame_index: fi }),
          visual: s.visual ? { ...s.visual, status: 'ready', image_url: url, frame_index: fi } : null
        });
      },
      onQuestionReady: (data) => {
        const sid = String(data.section_id ?? '');
        const qid = String(data.question_id ?? '');
        const diff = String(data.difficulty ?? 'warm');
        const pdata = data.data as Record<string, unknown> ?? {};
        canvas = canvas.map(s => s.id !== sid ? s : {
          ...s,
          mergedFields: mergePracticeProblem(s.mergedFields, qid, diff, pdata)
        });
      },
      onResourceFinalised: () => {
        complete = true;
      },
      onError: () => { complete = true; }
    });
  });
</script>

<div data-generation-complete={complete ? 'true' : undefined} class="print-canvas">
  <V3Canvas sections={canvas} stage={complete ? 'complete' : 'generating'}
    templateId="diagram-led" />
</div>
```

```python
@v3_studio_router.get("/generations/{generation_id}/blueprint")
async def get_v3_generation_blueprint(
    generation_id: str,
    current_user: User = Depends(get_current_user),
):
    owner = await v3_studio_store.get_generation_owner(generation_id)
    if owner != current_user.id:

    stored = await v3_studio_store.get_blueprint_for_generation(generation_id)

        blueprint_id=generation_id,
        blueprint=stored.blueprint,
        template_id=stored.template_id,
    )
```

Add the handler and state:

```typescript
let pdfLoading = $state(false);
let pdfError = $state<string | null>(null);

async function handleDownloadPdf() {
  const blueprint = v3Studio.blueprint;
  if (!blueprint) return;
  pdfLoading = true;
  pdfError = null;
  try {
    await downloadV3GenerationPdf(blueprint.blueprint_id, {
      school_name: '',
      teacher_name: '',
      include_toc: false,
      include_answers: true,
    });
  } catch (err) {
    pdfError = friendly(err);
  } finally {
    pdfLoading = false;
  }
}
```

Add `downloadV3GenerationPdf` to `frontend/src/lib/api/v3.ts`:

```typescript
export async function downloadV3GenerationPdf(
  generationId: string,
  request: { school_name: string; teacher_name: string; include_toc: boolean; include_answers: boolean }
): Promise<void> {
  const res = await apiFetch(
    `/api/v1/v3/generations/${encodeURIComponent(generationId)}/export/pdf`,
    { method: 'POST', headers: bearerHeaders(), body: JSON.stringify(request) }
  );
  await ensureOk(res, 'Failed to export PDF.');
  const blob = await res.blob();
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `lesson-${generationId}.pdf`;
  a.click();
  URL.revokeObjectURL(url);
}
```
