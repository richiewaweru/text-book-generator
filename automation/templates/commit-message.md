# Commit Message Template

Use Conventional Commits with one of these types:
- `feat`
- `fix`
- `refactor`
- `docs`
- `test`
- `chore`
- `ci`
- `build`

## Format
`type(scope): concise summary`

## Examples
- `feat(backend): add planner retry metrics`
- `docs(engineering): document release tagging flow`
- `ci(repo): add architecture guard workflow`
